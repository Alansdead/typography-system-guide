Implementation Guidelines for iOS

Here are the best practices to properly implement typography in iOS applications:

  • Use San Francisco font for a native iOS look and feel
  • Implement Dynamic Type for accessibility compliance
  • Set minimum font size for body text to 17pt for readability
  • For custom fonts, use UIFontMetrics for Dynamic Type
  • Test your app with all Dynamic Type size settings
// Swift code for Dynamic Type with system font
label.font = UIFont.preferredFont(forTextStyle: .body)
label.adjustsFontForContentSizeCategory = true

// With custom font
let customFont = UIFont(name: "CustomFont-Regular", size: 16)!
label.font = UIFontMetrics.default.scaledFont(for: customFont)
label.adjustsFontForContentSizeCategory = true

Implementation Guidelines for Android

Here are the best practices to properly implement typography in Android applications:

  • Always define text sizes in scalable pixels (sp) to respect user font size preferences
  • The minimum recommended size for body text is 16sp for readability
  • For dense lists or information, you can use 14sp with a slightly heavier weight
  • Use dynamic layouts with wrap_content and constraintLayout to adapt to different text sizes
  • Test your app with font scaling set to at least 1.3x to ensure layout adaptability
<!-- XML layout example -->
<TextView
  android:layout_width="match_parent"
  android:layout_height="wrap_content"
  android:textSize="16sp"
  android:textColor="@color/textPrimary"
  android:text="@string/sample_text" />

<!-- Compose example -->
Text(
  text = "Sample Text",
  style = MaterialTheme.typography.bodyLarge,
  modifier = Modifier.fillMaxWidth()
)

Typography Accessibility Guidelines â€“ WCAG 2.1 AA Compliance

Creating accessible typography is essential for ensuring your app can be used by everyone, regardless of their abilities. This guide follows WCAG Success Criterion 1.4.4 (Resize Text), 1.4.3 (Contrast), and related guidelines.

WCAG Success Criterion 1.4.4 - Resize Text (Level AA)

Requirement

Level AA

Requirement: Text can be resized without assistive technology up to 200 percent without loss of content or functionality.

Mobile Application Interpretation

For mobile applications, this criterion is met when:

  • Apps respect the operating system's text size preferences
  • Text scaling is implemented without relying solely on zoom functionality
  • Content remains accessible and functional at larger text sizes
  • Layout adapts to accommodate scaled text without horizontal scrolling

Assistive Technology vs. Accessibility Features

  • Distinguish between OS accessibility features (such as larger text) and screen readers
  • Test with both features enabled to ensure compatibility

WCAG Success Criterion 1.4.3 - Contrast (Level AA)

Requirement

Level AA

Text and images of text have a contrast ratio of at least 4.5:1, except for large text, incidental text, and logotypes.

Contrast Best Practices

  • Test all foreground/background color pairs in your app for compliance
  • Use tools such as the WebAIM Contrast Checker
  • Avoid using color alone to convey information
