Implementation Guidelines for iOS

Here are the best practices to properly implement typography in iOS applications:

  • Use the San Francisco font for a native iOS look and feel
  • Implement Dynamic Type for accessibility compliance
  • Set the minimum font size for body text to 17pt for readability
  • For custom fonts, use UIFontMetrics to scale appropriately with Dynamic Type
  • Test your app with all Dynamic Type size settings to ensure the layout adapts correctly
// Swift code for implementing Dynamic Type with system font
label.font = UIFont.preferredFont(forTextStyle: .body)
label.adjustsFontForContentSizeCategory = true

// Swift code for implementing Dynamic Type with custom font
let customFont = UIFont(name: "CustomFont-Regular", size: 16)!
label.font = UIFontMetrics.default.scaledFont(for: customFont)
label.adjustsFontForContentSizeCategory = true

Android allows users to adjust their font size through system settings. Your app should respect these settings for accessibility. Text elements defined in scalable pixels (sp) will automatically scale with user-preferred font size.

Implementation Guidelines for Android

Here are the best practices to properly implement typography in Android applications:

  • Always define text sizes in scalable pixels (sp) to respect user font size preferences
  • The minimum recommended size for body text is 16sp for readability
  • For dense lists or information, you can use 14sp with a slightly heavier weight
  • Use dynamic layouts with wrap_content and constraintLayout to adapt to different text sizes
  • Test your app with font scaling set to at least 1.3x to ensure layout adaptability
<!-- XML layout example -->
<TextView
  android:layout_width="match_parent"
  android:layout_height="wrap_content"
  android:textSize="16sp"
  android:textColor="@color/textPrimary"
  android:text="@string/sample_text" />

<!-- Compose example -->
Text(
  text = "Sample Text",
  style = MaterialTheme.typography.bodyLarge,
  modifier = Modifier.fillMaxWidth()
)

Typography Accessibility Guidelines - WCAG 2.1 AA Compliance

Creating accessible typography is essential for ensuring your app can be used by everyone, regardless of their abilities. This guide follows WCAG Success Criterion 1.4.4 (Resize Text - Level AA), which requires that text can be resized up to 200% without loss of content or functionality.

WCAG Success Criterion 1.4.4 - Resize Text (Level AA)

Requirement: Text can be resized without assistive technology up to 200 percent without loss of content or functionality.

Mobile Application Interpretation

For mobile applications, this criterion is met when:

  • Apps respect the operating system's text size preferences
  • Text scaling is implemented without relying solely on zoom functionality
  • Content remains accessible and functional at larger text sizes
  • Layout adapts to accommodate scaled text without horizontal scrolling

Assistive Technology vs. Accessibility Features

✅ Acceptable (Accessibility Features)
  • iOS: Dynamic Type size adjustments in Settings
  • Android: Font size scaling in Display settings
  • Both: Bold text system preferences
  • WebViews: Built-in browser zoom functionality
⚠️ Not Sufficient Alone (Assistive Technology)
  • iOS: Magnifier or 3-finger zoom
  • Android: Magnification gestures
  • Both: External screen reader zoom features

Note: While these are valuable, they should supplement, not replace, proper text scaling support.

WebView Accessibility Considerations

For apps using WebViews, additional considerations apply:

WebView Text Scaling Options

  • Browser Zoom: Allow standard pinch-to-zoom functionality
  • Text Scaling: Respect system font size preferences where possible
  • Custom Implementation: Provide app-specific text scaling controls
// iOS WebView scaling
extension WKWebView {
var zoomScale: Double {
get {
guard let scale = value(forKey: "viewScale") as? Double else { return 1.0 }
return scale
}
set { setValue(newValue, forKey: "viewScale") }
}
}
// Android WebView scaling
fun WebView.setScale(initialScale: Float, zoomScale: Float) {
val zoom = (initialScale * zoomScale * 100).toInt()
setInitialScale(zoom)
}

Best Practices Summary

Design Phase

  • Design layouts that accommodate 200%+ text scaling
  • Use minimum 16sp/17pt for body text
  • Plan for content reflow and wrapping
  • Consider scrollable containers for complex layouts

Development Phase

  • Implement Dynamic Type (iOS) and sp units (Android)
  • Use flexible layout systems
  • Test with maximum accessibility settings
  • Handle font size change notifications

Testing Phase

  • Test with real users who rely on large text
  • Use accessibility testing tools
  • Verify functionality remains intact at all sizes
  • Document any limitations transparently
