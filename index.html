Implementation Guidelines for iOS

Here are the best practices to properly implement typography in iOS applications:

  • Use the San Francisco font for a native iOS look and feel
  • Implement Dynamic Type for accessibility compliance
  • Set the minimum font size for body text to 17pt for readability
  • For custom fonts, use UIFontMetrics to scale appropriately with Dynamic Type
  • Test your app with all Dynamic Type size settings to ensure the layout adapts correctly
// Swift code for implementing Dynamic Type with system font label.font = UIFont.preferredFont(forTextStyle: .body) label.adjustsFontForContentSizeCategory = true // Swift code for implementing Dynamic Type with custom font let customFont = UIFont(name: "CustomFont-Regular", size: 16)! label.font = UIFontMetrics.default.scaledFont(for: customFont) label.adjustsFontForContentSizeCategory = true

Implementation Guidelines for Android

Here are the best practices to properly implement typography in Android applications:

  • Always define text sizes in scalable pixels (sp) to respect user font size preferences
  • The minimum recommended size for body text is 16sp for readability
  • For dense lists or information, you can use 14sp with a slightly heavier weight
  • Use dynamic layouts with wrap_content and constraintLayout to adapt to different text sizes
  • Test your app with font scaling set to at least 1.3x to ensure layout adaptability
<TextView android:layout_width="match_parent" android:layout_height="wrap_content" android:textSize="16sp" android:textColor="@color/textPrimary" android:text="@string/sample_text" /> Text( text = "Sample Text", style = MaterialTheme.typography.bodyLarge, modifier = Modifier.fillMaxWidth() )

Typography Accessibility Guidelines

Creating accessible typography is essential for ensuring your app can be used by everyone, regardless of their abilities. Both iOS and Android provide tools to make your typography accessible.

Font Size Accessibility

About 25-33% of mobile users change their default font size settings, with most opting for larger text sizes. Supporting these preferences is crucial for accessibility.

Dynamic Type (iOS)

iOS uses Dynamic Type to allow users to adjust text size system-wide. There are several size categories from xSmall to AX5 (largest accessibility size).

Platform-Specific Accessibility Features

iOS Dynamic Type

iOS Dynamic Type allows text to automatically adjust based on user preferences.

Key implementation tips:

  • Use UIFont.preferredFont(forTextStyle:) for system fonts
  • Use UIFontMetrics for custom fonts
  • Set adjustsFontForContentSizeCategory to true
  • Listen for UIContentSizeCategory.didChangeNotification to update UI when settings change
Android Font Scaling

Android Font scaling allows users to adjust text size system-wide.

Key implementation tips:

  • Always use sp (scaled pixels) for text sizes
  • Implement flexible layouts with wrap_content
  • Use ConstraintLayout for responsive designs
  • Consider adding scrolling to containers to prevent text clipping
  • Test with font scale set to at least 1.3x

Best Practices for Accessible Typography

Regardless of platform, follow these guidelines to ensure your typography is accessible:

  1. Minimum Text Size: Use at least 16pt/sp for body text (iOS recommends 17pt minimum)
  2. Contrast Ratio: Maintain a minimum contrast ratio of 4.5:1 between text and background for normal text, and 3:1 for large text
  3. Line Height: Use line heights between 120-160% of the font size for optimal readability
  4. Flexible Layouts: Design layouts that adapt to different text sizes without breaking
  5. Scrollable Content: Make content containers scrollable to prevent clipping when text is enlarged
  6. Text Overflow: Handle text overflow gracefully, using ellipsis or wrapping
  7. Test with Accessibility Tools: Use iOS Accessibility Inspector and Android Accessibility Scanner

Special Considerations for Custom Fonts

If you need to use custom fonts in your app, keep these additional points in mind:

  • iOS: Use UIFontMetrics.default.scaledFont(for:) to make custom fonts scale with Dynamic Type
  • Android: Always specify text sizes in sp, even with custom fonts
  • Test your custom font thoroughly at different sizes - some fonts become harder to read at smaller sizes
  • Consider using a custom font with multiple weights to maintain readability across sizes
  • Avoid overly decorative fonts for body text or important information
