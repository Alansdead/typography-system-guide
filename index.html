Implementation Guidelines for iOS

  • Use San Francisco font for a native iOS look and feel
  • Implement Dynamic Type for accessibility compliance
  • Set minimum font size for body text to 17pt for readability
  • For custom fonts, use UIFontMetrics for Dynamic Type
  • Test your app with all Dynamic Type size settings
// Swift code for Dynamic Type with system font
label.font = UIFont.preferredFont(forTextStyle: .body)
label.adjustsFontForContentSizeCategory = true

// With custom font
let customFont = UIFont(name: "CustomFont-Regular", size: 16)!
label.font = UIFontMetrics.default.scaledFont(for: customFont)
label.adjustsFontForContentSizeCategory = true

Implementation Guidelines for Android

  • Use "sp" units for all text sizes
  • Minimum recommended body text size: 16sp
  • Use constraint layouts for adaptive design
  • Test with font scaling set to at least 1.3x
<!-- XML layout example -->
<TextView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:textSize="16sp"
    android:textColor="@color/textPrimary"
    android:text="@string/sample_text" />

<!-- Compose example -->
Text(
    text = "Sample Text",
    style = MaterialTheme.typography.bodyLarge,
    modifier = Modifier.fillMaxWidth()
)

Testing for WCAG 1.4.4 Compliance

iOS Testing

  • Test with all Dynamic Type sizes (XS to AX5)
  • Verify text scales appropriately without layout breaks
  • Check that all text remains readable and accessible
  • Ensure no horizontal scrolling is required
  • Test with Bold Text setting enabled
  • Verify Large Content Viewer works for constrained spaces
  • Test with VoiceOver enabled
  • Test in both Light and Dark modes

Android Testing

  • Test with font scale from 0.85x to 1.3x
  • Use ADB to test extreme scaling (up to 2.0x): adb shell settings put system font_scale 2.0
  • Verify layouts adapt without content loss
  • Check that sp units are used consistently
  • Test with different display sizes
  • Test with TalkBack enabled
  • Test with system Bold Text and High Contrast Text

WebView Accessibility Considerations

  • Allow pinch-to-zoom and system text scaling in WebViews
  • Test with font scaling and zoom enabled
// iOS WebView scaling
extension WKWebView {
    var zoomScale: Double {
        get {
            guard let scale = value(forKey: "viewScale") as? Double else { return 1.0 }
            return scale
        }
        set { setValue(newValue, forKey: "viewScale") }
    }
}
// Android WebView scaling
fun WebView.setScale(initialScale: Float, zoomScale: Float) {
    val zoom = (initialScale * zoomScale * 100).toInt()
    setInitialScale(zoom)
}

Best Practices Summary

Design Phase

  • Design layouts that accommodate 200%+ text scaling
  • Use minimum 16sp/17pt for body text
  • Plan for content reflow and wrapping
  • Consider scrollable containers for complex layouts
  • Ensure color contrast meets WCAG 2.1 AA

Development Phase

  • Implement Dynamic Type (iOS) and sp units (Android)
  • Use flexible layout systems
  • Test with maximum accessibility settings
  • Handle font size change notifications
  • Use semantic HTML (for web content)

Testing Phase

  • Test with real users who rely on large text
  • Use accessibility testing tools
  • Verify functionality remains intact at all sizes
  • Document any limitations transparently
  • Provide feedback mechanisms for users
